# signal-newsletter

> Automated personal tech digest — generated by Claude Sonnet with live web search, delivered every Friday morning.

Built on GitHub Actions + Anthropic API + nodemailer over OVH SMTP. Zero infrastructure, ~$0.02–0.04 per run.

---

## How it works

Every Friday at 07:00 CET, a GitHub Actions workflow:

1. Calls Claude Sonnet with the `web_search` tool to research the past 7 days
2. Generates a structured JSON newsletter with 4–5 curated items
3. Verifies image URLs via HEAD requests before rendering
4. Builds a dark-mode HTML email with per-category accent colors
5. Ships it over OVH SMTP
6. Commits `history.json` to the repo — used next week to avoid repeating topics

On the **first Friday of each month**, a monthly digest is also generated — Claude synthesizes the last 4 editions into 3 macro trends, no web search, pure analysis.

If anything fails, an alert email is sent immediately with a direct link to the failed run.

Each weekly item includes: editorial intro, summary with intentional bold + highlight markup, verified source links with publication dates, a "Signal" takeaway block, and a deep-link to Claude pre-loaded with context.

## Stack

| Layer | Choice |
|---|---|
| Runtime | Node.js 20 (ESM) |
| LLM | `claude-sonnet-4-6` via `@anthropic-ai/sdk` |
| Web search | Anthropic `web_search_20250305` tool |
| Email | nodemailer → OVH SMTP (ssl0.ovh.net:465) |
| Scheduler | GitHub Actions cron |
| History | `history.json` committed to repo (last 8 editions) |

## Setup

### 1. Clone and push

```bash
git clone <this-repo> && cd signal-newsletter
gh repo create signal-newsletter --private --push --source=.
```

### 2. Set repository secrets

`Settings → Secrets and variables → Actions → New repository secret`

| Secret | Value |
|---|---|
| `ANTHROPIC_API_KEY` | API key from [console.anthropic.com](https://console.anthropic.com/settings/keys) |
| `SENDER_EMAIL` | Sending email address |
| `SENDER_PASSWORD` | SMTP password for that address |

### 3. Trigger manually to verify

```bash
gh workflow run newsletter.yml --repo <your-username>/signal-newsletter
gh run list --repo <your-username>/signal-newsletter
```

Check your inbox. Done.

## Customization

All personalization lives in `SYSTEM_PROMPT` inside `src/generate.js`:

- **Topics** — edit the themes list to match your interests
- **Trusted sources** — curated allowlist of domains, no AI-generated content
- **Markup rules** — strict bold/highlight constraints to keep formatting intentional
- **Recipient** — update `RECIPIENT` at the top of each file

The `history.json` file is auto-managed. Don't edit it manually.
